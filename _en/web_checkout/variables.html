Variables to be sent to the gateway

Field Type Size Description Required in
amount Numeric 14 The total value of the transaction. It can have two decimal places. Ej. 10000.00 ó 10000. Every country
merchantId Numeric 12 This is the identifier number of the merchant in the PAYU system. You can find this number in your PayU Biz account. Configuration menu, Technical information section. Every country
referenceCode Alfanumeric 255 This is the reference sale or order. It must be unique for each transaction that is sent to the system. Usually, this is a way to identify requests that are sent to the payment gateway. Every country
accountId Numeric 6 The identifier of the user account for each partner country of sale. Displays payment methods which are only available in that country. Every country
description Alfanumeric 255 The description of the sale. Every country
tax Numeric 14,2 This is the transaction VAT. If VAT zero is sent the system, 19% will be applied automatically. It can contain two decimals. Eg 19000.00. In the where you do not charge VAT, it should should be set as 0. Every country
taxReturnBase Numeric 14,2 This is the base value upon which VAT is calculated. If you do not charge VAT, it should be sent as 0. Every country
signature Alfanumeric 32 The digital signature which is created for each transaction. Every country
currency Numeric 3 The currency in which the payment is made. The reconciliation process is made in pesos at the representative rate of that day. See supported currencies. Every country
test Numeric 1 Variable used to test credit cards. Values can be 1 or 0. —
buyerEmail Alfanumeric 255 The field which contains the email of the buyer. This is used to notify they buyer of the result of the transaction by email. Validation is recommended if you include this field in your payment form. Every country
