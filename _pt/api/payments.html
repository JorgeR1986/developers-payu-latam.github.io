Os métodos disponíveis para o API de pagamentos são: PING, Consulta de medios de pagos activos, e Procesamiento de transacciones. A seguir você encontrará a descrição de cada um. 
Processamento de transações
Contamos com diversos métodos para o processamento de transações:

 
 
Todas as operações disponíveis são:
  • Autorização: Usada para verificar se um cartão de crédito se encontra ativo, se possui crédito etc. A transação não é finalizada até que seja enviada uma transação de captura (somente disponível para contas do Brasil).
  • Captura: Finaliza uma transação previamente autorizada. É aqui que se realiza o débito da conta do cartão (somente disponível para contas do Brasil).
  • Autorização e captura: Este é o tipo de transação mais usada. Com esta opção se envia a autorização, o montante da transação e se for aprovada imediatamente é realizada a captura..
  • Anulação: Usada para reverter uma transação previamente autorizada (somente disponível para contas do Brasil).
  • Reembolso: Usada para reverter uma transação previamente capturada.

6. Quando o pagador finalizar a transação no escritório virtual de seu banco, será direcionado para a página de resposta que está configurada no módulo administrativo e deve ser mostrada a seguinte informação: Os campos com o caractere "$" no início, são parâmetros enviados por GET e que devem capturar.

Nota:
  1. A página deve ter um link de “nova tentativa” que permitirá ao pagador tentar o pagamento novamente se a transação não foi bem-sucedida.
  2. A página deve ter um link para “finalizar a transação”, o qual direcione o pagador para um local de seu site.
  3. Você deve incluir uma opção para imprimir um comprovante de pagamento caso o pagador deseje realizar uma reclamação.
  4. O campo “Status” deve conter a informação de acordo com a seguinte tabela. deve-se rever as variáveis polTransactionState e polResponseCode dependendo da combinação, escolhe-se o texto a ser mostrado na tabela.
polTransactionState polResponseCode Estado
4 1 Transação aprovada
6 5 Transação falhada
6 4 Transação rejeitada
12 ou 14 9994 ou 25 Transação pendente, por favor rever se o débito foi realizado junto ao banco.
 
Considerar:
  1. As variáveis obrigatórias transaction.order.signature e transaction.deviceSessionId precisam de alguns passos adicionais, para saber mais sobre seu funcionamento acesse a página: Considerações sobre as variáveis.
  2. A variável transaction.threeDomainSecure ão substitui as informações do cartão ou os campos necessários para iniciar uma transação. Este objeto é adicional.
  3. A variável transaction.threeDomainSecure corresponde a um cenárioo Pass Through no qual o merchant realiza a autenticação 3DS por conta própria. Atualmente, este cenário está disponível só para transações na Colômbia e no Brasil.
  4. Se desejar confirmar o status de uma transação você conta com duas opções:
    A. Com a página de confirmação enviada na variável “transaction.notifyUrl” ou a configurada no módulo administrativo na seção configurar integração, e desejar saber mais sobre a página de confirmação acesse Integração Webcheckout
    B. Com a API de consultas, que permitirá programar uma tarefa de consulta do status das transações em um intervalo de tempo determinado, para mais informações acesse a página API de Consultas.
 
Nota: Em algumas variáveis são acrescentados os termos “Payer” e “Buyer” (comprador e pagador, respectivamente). O comprador é a pessoa que deseja adquirir um produto ou serviço, enquanto que o pagador é a pessoa que realiza o pagamento em dinheiro. Em alguns casos os dados do comprador e do pagador podem diferir já que a pessoa que ordena a compra através do web site não é a mesma que realiza o pagamento
Exemplo: Um jovem deseja adquirir um videogame através da internet (comprador), mas no momento de realizar o pagamento seu pai insere os dados de seu cartão de crédito (pagador).
 
Logos dos meios de pagamento: Se desejar incluir os logos das formas de pagamento disponíveis no seu web site, você pode acessar http://www.payulatam.com/logos/ e fazer o download das imagens de cada país.
