    Métodos de pagos con la API PayU ofrece una API que le permite a tu comercio procesar diferentes tipos de transacciones con múltiples medios de pago dependiendo el país seleccionado. Si deseas integrarte con nuestra API deberás apuntar las peticiones a las siguientes URLs:

Pruebas: https://sandbox.api.payulatam.com/payments-api/4.0/service.cgi
Producción: https://api.payulatam.com/payments-api/4.0/service.cgi
 
 
Los métodos disponibles para la API de pagos son: PING, CONSULTA DE MEDIOS DE PAGOS ACTIVOS, y PROCESAMIENTO DE TRANSACCIONES. A continuación encontrarás la descripción de cada uno.  
 
Procesamiento de transacciones
Contamos con diferentes métodos para el procesamiento de transacciones:

 
 
Todas las operaciones disponibles son:
  • Autorización: se usa para verificar si una tarjeta de crédito se encuentra activa, si posee fondos, etc. La transacción no se finaliza hasta que se envíe una transacción de captura.
  • Captura: finaliza una transacción previamente autorizada. Es aquí cuando se realiza el débito de la cuenta de la tarjeta.
  • Autorización y captura: este es el tipo de transacción más usada. Con esta opción se envía a autorización el monto de la transacción y si es aprobada, inmediatamente se realiza la captura.
  • Anulación: Se usa para reversar una transacción previamente autorizada.
  • Reembolso: Se usa para reversar una transacción previamente capturada.

    • 1. El precio de contado


    2. El precio total financiado


    3. La cantidad de cuotas y monto de cada una


    4. La tasa de interés efectiva anual aplicada (TEA)


    5. El costo financiero total (CFT)



  • La información del costo financiero total de la operación deberá tener las siguientes características:

    • 1. Estar ubicado de forma contigua al resto de las variables informadas

    2. Aparecer en una tipografía en color destacado, de idéntica fuente y con un tamaño al menos cinco veces mayor al utilizado para informar la tasa de interés efectiva anual y/o la cantidad de cuotas y/o su importe.

  1. Cuotas soportadas para Tarjetas Débito Visa: 1.
  2. El código de seguridad de las Tarjetas Débito Visa es de 3 dígitos.
  3. Temporalmente las Tarjetas Débito Visa solo pueden ser probadas en el ambiente de producción.
  4. Cuotas soportadas para Crédito Fácil Codensa: 1 a 12, 18, 24, 36 y 48.
  5. Se debe enviar el tipo de documento del pagador (dniType) para Crédito Fácil Codensa.
Tipo de documento
En el proceso de pago se le debe permitir escoger al pagador el tipo de documento, el cual se debe enviar con formato ISO en el nodo payer.dniType. Los códigos ISO admitidos son:
ISO Descripción
CC Cédula de ciudadanía.
CE Cédula de extranjería.
NIT Número de Identificación Tributario.
TI Tarjeta de Identidad.
PP Pasaporte.
IDC Identificador único de cliente, para el caso de ID’s únicos de clientes/usuarios de servicios públicos.
CEL En caso de identificarse a través de la línea del móvil.
RC Registro civil de nacimiento.
DE Documento de identificación extranjero.
