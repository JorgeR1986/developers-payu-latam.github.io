El producto Tokenización te permite almacenar los datos de las tarjetas de crédito de tus clientes de forma segura, por medio de la creación de un token con el cuál podrás realizar cobros periódicamente o implementar la funcionalidad de pago en 1 Clic. Siguiendo así los estándares de seguridad en la manipulación de los datos de tarjetas de crédito PCI DSS (Payment Card Industry Data Security Standard).
Para el uso de las funcionalidades del producto de Tokenización desde el API, debes tener habilitado este servicio en tu cuenta PayU, si deseas adquirirlo comunícate con sac@payulatam.com.  
 
¿Cómo funciona?  
El producto “Tokenizacion” incluye dos procesos independientes:

 
 
Los métodos disponibles para la tokenización son: Registro individual de tarjeta de crédito, Registro masivo de tarjetas de crédito, Cobro individual con token, Cobros masivo con token, Consulta de token y Eliminación de token. A continuación encontrarás la descripción de cada uno.

Registro masivo de tarjetas de crédito
 
Mediante esta funcionalidad puedes registrar los datos de las tarjetas de crédito de varios clientes y así obtener los códigos secuenciales token.
Para tokenizar tarjetas de crédito masivamente debes crear un archivo con formato CSV bajo las siguientes condiciones:  
  1. Cada registro de tarjeta de crédito a tokenizar, debe contener los datos para el registro de una tarjeta de crédito separados por coma en el siguiente orden: ID Pagador, nombres y apellidos, número de tarjeta de crédito, fecha de expiración, franquicia, y número de identificación.
  2. El archivo no debe tener encabezado. En la primera línea va el primer registro.
  3. El archivo debe estar codificado bajo el estándar UTF-8.
  4. El archivo no debe tener más de 10.000 registros.
 

  • JSON
  • XML
Ejemplo petición: 
{
   "language": "es",
   "command": "CREATE_BATCH_TOKENS",
   "merchant": {
      "apiLogin": "pRRXKOl8ikMmt9u",
      "apiKey": "4Vj8eK4rloUd272L48hsrarnUA"
   },
   "contentFile": "MDAxLG5vbWJyZSBhcGVsbGlkbyAxLDQxMTExMTExMTExMTExMTEsMjAxNS8wMSxWSVNBLDEyMzQ1Ng=="
}
 
Ejemplo respuesta: 
{
   "code": "SUCCESS",
   "error": null,
   "id": "edd187bb-067c-4016-b94d-3a4727cc86eb"
}
 
Debes implementar una funcionalidad que tome el contenido del archivo y lo pase a base 64, el resultado de este procedimiento es el que se envía en el parámetro <contentFile>.
Ejemplo petición: 
<request>
   <language>es</language>
   <command>CREATE_BATCH_TOKENS</command>
   <merchant>
      <apiLogin>pRRXKOl8ikMmt9u</apiLogin>
      <apiKey>4Vj8eK4rloUd272L48hsrarnUA</apiKey>
   </merchant>
   <contentFile>MDAxLG5vbWJyZSBhcGVsbGlkbyAxLDQxMTExMTExMTExMTExMTEsMjAxNS8wMSxWSVNBLDEyMzQ1Ng==</contentFile>
</request>

<creditCardTokenBatchResponse>
   <code>SUCCESS</code>
   <id>edd187bb-067c-4016-b94d-3a4727cc86eb</id>
</creditCardTokenBatchResponse>

Cobro individual con token
 
Esta funcionalidad te permite realizar cobros, utilizando un código Token que fue previamente creado por nuestro sistema, y con el cual se almacenaron los datos de las tarjetas de crédito de tus clientes de forma segura.
Nota:
  1. Para procesar con código de seguridad CVV es necesario envíar los parámetros creditCard.securityCode con el valor del CVV y creditCard.processWithoutCvv2 con el valor false en la petición.

  • Brasil
  • Colombia
  • México
  • Perú

Cobros masivo con token
 
Esta funcionalidad te permitirá realizar cobros masivos, utilizando los Token que fueron previamente creados por nuestro sistema
Para esta funcionalidad debes crear un archivo con formato CSV bajo las siguientes condiciones:  
  1. Cada registro de transacción a procesar, debe contener los datos para su procesamiento separados por coma y en el siguiente orden: ID Cuenta (identificador de la cuenta virtual en PayU), Token, código de seguridad, número de cuotas, referencia de venta, descripción de la venta, email del comprador, moneda (codificación ISO), valor total (impuesto incluído), impuesto, valor base devolución, valor adicional, idioma (codificación ISO).
  2. El archivo no debe tener encabezado. En la primera línea va el primer registro.
  3. El archivo debe estar codificado bajo el estándar UTF-8.
  4. El archivo no debe tener más de 10.000 registros.

Eliminación masiva de tokens
 
Esta funcionalidad te permitirá eliminar masivamente los tokens que fueron creados previamente en el sistema.
Para esta funcionalidad debes crear un archivo con formato CSV bajo las siguientes condiciones:  
  1. Cada registro de token a eliminar, debe contener los datos para su procesamiento separados por coma y en el siguiente orden: PayerId, TokenId.
  2. El archivo no debe tener encabezado. En la primera línea va el primer registro.
  3. El archivo debe estar codificado bajo el estándar UTF-8.
  4. El archivo no debe tener más de 10.000 registros.
  5. Debes implementar una funcionalidad que tome el contenido del archivo y lo pase a base 64, el resultado de este procedimiento es el que se envía en el parámetro <contentFile>.
 

  • JSON
  • XML
Ejemplo petición: 
{
   "language": "es",
   "command": "REMOVE_BATCH_TOKENS",
   "merchant": {
      "apiLogin": "pRRXKOl8ikMmt9u",
      "apiKey": "4Vj8eK4rloUd272L48hsrarnUA"
   },
   "contentFile": "UGF5ZXJJZDEsYWQ4NGQ2NzEtYjZiOC00YjEyLWFkNTktZmYxZDJhNjQ0M2NhDQpQYXllcklkMiw0ZGYxNjMwYy03MDkyLTRhNjgtODE3MC0yYzI2YzZjOTUyMDg="
}
 
Ejemplo respuesta: 
{
   "code": "SUCCESS",
   "error": null,
   "id": "37233713-9c0c-49ae-b412-30e6e0ddabf4"
}
 
Ejemplo petición:

   <command>REMOVE_BATCH_TOKENS</command>
   <merchant>
      <apiLogin>pRRXKOl8ikMmt9u</apiLogin>

   </merchant>
