Métodos de pago con la api usando SDK  
PayU ofrece una API que le permite a tu comercio procesar diferentes tipos de transacciones con múltiples medios de pago dependiendo el país seleccionado.  
Si deseas integrarte con nuestra API deberás apuntar las peticiones a las siguientes URLs:  
  • SDK Java
  • SDK PHP
PayU.paymentsUrl = “https://api.payulatam.com/payments-api/”;
PayU.reportsUrl = “https://api.payulatam.com/reports-api/”;
Environment::setPaymentsCustomUrl(“https://api.payulatam.com/payments-api/4.0/service.cgi”);
Environment::setReportsCustomUrl(“https://api.payulatam.com/reports-api/4.0/service.cgi”);
Environment::setSubscriptionsCustomUrl(“https://api.payulatam.com/payments-api/rest/v4.9/”);
 
Los métodos disponibles para la API de pagos son: PING, CONSULTA DE MEDIOS DE PAGOS ACTIVOS, y PROCESAMIENTO DE TRANSACCIONES. A continuación encontrarás la descripción de cada uno.
 
 
 
Procesamiento de transacciones
 
Contamos con diferentes métodos para el procesamiento de transacciones:

Todas las operaciones disponibles son:

  • Autorización: se usa para verificar si una tarjeta de crédito se encuentra activa, si posee fondos, etc. La transacción no se finaliza hasta que se envíe una transacción de captura (solo disponible para cuentas de Brasil).
  • Captura: finaliza una transacción previamente autorizada. Es aquí cuando se realiza el débito de la cuenta de la tarjeta (solo disponible para cuentas de Brasil).
  • Autorización y captura: este es el tipo de transacción más usada. Con esta opción se envía a autorización el monto de la transacción y se es aprobada inmediatamente se realiza la captura.
  • Anulación: Se usa para reversar una transacción previamente autorizada (solo disponible para cuentas de Brasil).
  • Reembolso: Se usa para reversar una transacción previamente capturada.
